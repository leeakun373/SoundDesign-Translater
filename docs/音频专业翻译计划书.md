# 音频专业翻译系统 — 计划书

> 版本：v1.0  
> 项目：`SoundDesign Translater`（详见 [开发历史.md](./开发历史.md)）  
> 目标：在现有 NLLB 本地翻译基础上，叠加 UCS/音效领域术语层，支持描述翻译、库文件名翻译、多工具共用。

---

## 一、背景与目标

### 1.1 现状

- 已有：`engine.py` + `app.py`，NLLB int8 本地中英互译，GUI 可双击启动。
- 问题：通用模型对音效命名、UCS 术语、库文件名（含 CatID/厂商码）翻译不稳定。

### 1.2 三个使用目的（均需支持）

| 编号 | 目的 | 典型输入 |
|------|------|----------|
| ① | 翻译描述/说明 | 「今天录了医院房间底噪，有点空调 hum」 |
| ② | 理解/翻译音效库文件名 | `ICEFric_Ice Axe Scratch Friction Ice Brick 01_ASO_Rec_CO100K` |
| ③ | UCSRenamer / Reaper 等共用 | 程序调用同一套术语与翻译逻辑 |

### 1.3 已确认的产品决策

| 决策项 | 选择 |
|--------|------|
| 文件名输出格式 | **B：可读中文为主**（例：`ICEFric_冰斧 刮擦 摩擦 冰砖 01_ASO_Rec_CO100K`） |
| 术语人工维护入口 | **`glossary/user_overrides.csv`**（唯一手改主表） |
| 底层翻译引擎 | 保持 NLLB + CTranslate2，不更换 |
| 离线要求 | 全程本地，不依赖联网 |

## 三、三种翻译模式

### 3.1 模式说明

| 模式 | 触发 | 处理逻辑 |
|------|------|----------|
| **自动**（默认） | 用户不指定 | 启发式判断：含大量 `_`/CatID/全大写 → 文件名模式；否则句子模式 |
| **句子** | 用户选择 | 整句 NLLB + 术语保护/纠偏 |
| **文件名** | 用户选择 | 拆段 → 分类 → 逐段查表/NLLB → 按 B 格式拼回 |

### 3.2 文件名模式 — 输出格式 B（已确认）

**原则：结构保留，可读词变中文，专业码不动。**

示例：

```
输入：ICEFric_Ice Axe Scratch Friction Ice Brick 01_ASO_Rec_CO100K

输出：ICEFric_冰斧 刮擦 摩擦 冰砖 01_ASO_Rec_CO100K
```

**分段规则：**

| 段类型 | 识别方式 | 处理方式 |
|--------|----------|----------|
| CatID | 匹配 UCS CatID 表（如 `ICEFric`） | **原样保留** |
| 序号 | 纯数字、`01`、`001` | **原样保留** |
| 厂商/项目码 | 配置表 + 启发式（如 `ASO`） | **默认原样保留** |
| 设备/录音码 | 如 `CO100K`、`Rec` | **默认原样保留**（`Rec` 可在 overrides 设特例） |
| 普通英文词 | 其余 | **术语表 → 无则 NLLB 单段** |

**分隔符：** 尽量保持输入中的 `_` 与空格结构；段内多个词译后仍用空格连接。

### 3.3 句子模式 — 示例

```
输入：木门 吱吱呀呀 厚重
输出：Wooden door creaking, heavy
      （术语表有对应词时替换为表内标准英文）

输入：Room tone of empty office
输出：空办公室的房间底噪
      （Room Tone 强制走术语表）
```

## 四、术语库设计

### 4.1 数据源与优先级

合并冲突时，**优先级从高到低**：

1. `glossary/user_overrides.csv` — 用户/AI 修正，**最终裁决**
2. `E:\Audio_Library\...\UCS\MyKeyWordV1.csv`
3. `E:\WorkSpace\UCSRenamer\data\ucs\ucs_catid_list.csv`
4. `E:\Audio_Library\...\UCS\Soundminer\_categorylist.csv`

### 4.2 词条类型（`term_type`）

| 类型 | 说明 | 翻译行为 |
|------|------|----------|
| `keyword` | 通用音效词（Whoosh, Room Tone） | 强制映射 |
| `category` | UCS 大类 | 强制映射 |
| `subcategory` | UCS 子类 | 强制映射 |
| `synonym` | UCS 同义词 | 映射到标准译法 |
| `catid` | UCS CatID | **never_translate** |
| `vendor` | 厂商/项目缩写 | **never_translate** |
| `device` | 录音设备码 | **never_translate** |
| `user_override` | 人工覆盖 | 最高优先级 |

### 4.3 合并产物

- **运行时库：** `glossary/audio_glossary.sqlite`
- **人工维护：** `glossary/user_overrides.csv`（见第五节）
- **重建命令：** `python glossary/build_glossary.py`

## 五、维护规范（`user_overrides.csv`）

### 5.1 文件位置（唯一手改入口）

```
LocalTranslateShared/glossary/user_overrides.csv
```

### 5.2 CSV 列定义

| 列名 | 必填 | 说明 |
|------|------|------|
| `en` | 是 | 英文原词或短语（大小写按惯例，匹配时忽略大小写） |
| `zh` | 视类型 | 中文译法；`never_translate` 类型可留空 |
| `term_type` | 是 | 见 4.2 |
| `action` | 是 | `translate` / `never_translate` |
| `note` | 否 | 备注，方便日后理解（如「与 Swish 区分」） |

### 5.3 示例

```csv
en,zh,term_type,action,note
Room Tone,房间底噪,keyword,translate,优先于机翻「房间音调」
Whoosh,呼啸声,keyword,translate,与 Swish(嗖嗖声) 区分
Swish,嗖嗖声,keyword,translate,
ICEFric,,catid,never_translate,UCS CatID 不译
ASO,,vendor,never_translate,厂商/项目码
Scratch,刮擦,synonym,translate,
```

### 5.4 改词后必做

```bash
cd LocalTranslateShared
python glossary/build_glossary.py
python -c "from glossary.matcher import smoke_test; smoke_test()"
```

GUI 重启或热加载后，用**原失败用例**回归。

## 六、翻译流水线（实现细节）

```
输入 text + mode(optional)
    │
    ├─ 空文本 → 返回 ""
    │
    ├─ detect_lang_pair()  → 中↔英
    │
    ├─ resolve_mode()      → auto | sentence | filename
    │
    ├─ [filename] parse_segments()
    │       └─ 每段 label: catid | code | term | phrase
    │
    ├─ protect_glossary()  → 已知词占位 ⟦G001⟧
    │
    ├─ NLLB translate_batch（仅 translatable 段/句子）
    │
    ├─ restore_glossary() + force_replace()
    │
    ├─ polish_audio_terms()  → 环境音→环境声 等
    │
    └─ 返回 TranslationResult { text, src, tgt, mode, direction_label }
```

### 6.1 GUI 新增控件（P3）

- 模式下拉：自动 / 句子 / 文件名
- 开关：「音频专业模式」（关 = 纯 NLLB，便于 A/B 对比）
- 状态栏：`文件名模式 · 中→英 · 专业开 · 命中术语 3 处`

## 七、分阶段实施计划

| 阶段 | 名称 | 交付 | 验收标准 |
|------|------|------|----------|
| **P0** | 术语地基 | `build_glossary.py`、`matcher.py`、`user_overrides.csv` 模板、SQLite | 合并三源成功；`Room Tone`→`房间底噪` 可查 |
| **P1** | 句子专业模式 | `engine` 接入 protect/restore/polish；GUI 专业开关 | 句子中 UCS 词不再乱译；可切纯 NLLB 对比 |
| **P2** | 文件名模式 B | `filename_parser.py`、拆段拼回、GUI 模式选择 | `ICEFric_Ice Axe...` 输出符合格式 B |
| **P3** | 体验完善 | 自动模式启发式、状态栏、对照信息 | 三种目的在 GUI 均可完成 |
| **P4** | 共用服务 | `service/server.py`、UCSRenamer 接入示例 | 同句翻译结果与 GUI 一致 |

**原则：每阶段结束即可独立使用，不阻塞后续。**

---

## 八、与 UCSRenamer 的关系

- **P0–P3：** LocalTranslateShared 独立可用，UCSRenamer 暂不改代码。
- **P4：** UCSRenamer 通过 `import` 或 `HTTP` 调用，**不再维护第二套术语规则**。
- 已有 `tools/translate_explanations.py` 的 `TERM_FIXES` 迁入 `glossary/polish.py`，避免双轨。

## 九、风险与对策

| 风险 | 对策 |
|------|------|
| 术语表错误导致强制错译 | 只收录确认词；`user_overrides` 可秒改；保留纯 NLLB 开关 |
| 文件名拆段误判 | 文件名模式手动选；CatID 表精确匹配优先 |
| 新库大量未收录词 | AI 批量提案 → 用户写入 overrides → rebuild |
| UCS 版本升级 | 重跑 `build_glossary.py`，overrides 不受影响 |
| 约束后不如原先 | A/B 开关对比；句子模式约束最轻，文件名模式约束最重 |

## 十、维护成本评估

| 项目 | 频率 | 操作 |
|------|------|------|
| 补/改一个词 | 偶尔 | 编辑 `user_overrides.csv` → rebuild → 回归 |
| UCS 大表更新 | 低 | 重跑 build，overrides 覆盖仍生效 |
| 挖 10 万行 DB 未覆盖词 | 可选一次 | AI 出清单，人工确认后批量写入 overrides |

**有 AI 协助时：** 用户只需描述「哪个输入、哪个词错了、想要什么结果」；AI 按 `.cursor/rules/glossary-maintenance.mdc` 改 CSV 并自测。
