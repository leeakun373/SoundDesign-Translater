# 补词批量验收 — 给另一个 AI 窗口的指令

## 背景

- **主表 `user_overrides.csv` 只有十几行是正常的**：它只放「你人工确认过的覆盖规则」，不是完整词典。
- **完整术语库**在 `glossary/audio_glossary.sqlite`（约 1 万+ 条），由 UCS 三源 + 主表自动合并生成。
- 主表作用：**最高优先级**，用来纠正底库译法、补充商业库高频词、标记 CatID/厂商码不译。

---

## 第一步：在本项目导出第 N 批（50 条）

```bash
# 需自备商业库，并指定路径
python tools/export_uncovered_batches.py --db path/to/ucs_assets.db --batch-index 1
```

输出文件：`glossary/batches/uncovered_batch_001.csv`（格式已统一，含去重）

CSV 列说明：

| 列 | 说明 |
|----|------|
| en | 英文词（已去重） |
| zh | **留空，由 AI 填写** |
| term_type | 已猜测：keyword / catid / vendor / device |
| action | translate 或 never_translate |
| note | 含 freq=出现次数 |
| count | 商业库出现次数 |
| review_status | pending → 填完后改为 approved / rejected |

---

## 第二步：复制下面整段给另一个 AI

```
你在做 LocalTranslateShared 项目的「音频术语补词」任务。

输入文件：e:\WorkSpace\LocalTranslateShared\glossary\batches\uncovered_batch_XXX.csv
（XXX 由用户指定，如 001）

请只做这一批（约 50 行），不要改其他文件。

任务：
1. 读取 CSV，对每一行填写 zh（中文译法）。
2. 校对 term_type 和 action：
   - 厂商缩写、CatID、设备码 → action=never_translate，zh 留空
   - 普通音效词 → action=translate，给出专业中文（参考 UCS/MyKeyWord 风格）
3. 去重：若 en 仅大小写不同，只保留一条。
4. 格式必须与 user_overrides.csv 一致，最终输出列：
   en,zh,term_type,action,note
5. note 写简短说明，如「freq=5000;批001;音效词」
6. 不确定的词条 note 加「待用户确认」，不要瞎编。
7. 输出为同目录新文件：uncovered_batch_XXX_filled.csv
8. 不要直接写入 user_overrides.csv；用户验收后再合并。

翻译风格：
- Room Tone → 房间底噪（不是房间音调）
- Whoosh / Swish 要区分
- Foley → 拟音
- 环境音 → 环境声
```

---

## 第三步：你验收

1. 打开 `uncovered_batch_XXX_filled.csv`，检查 zh / action 是否合理
2. 把 **approved** 的行追加到 `glossary/user_overrides.csv`（不要重复 en）
3. 运行：

```bash
build_glossary.bat
```

4. 用 GUI 或命令抽几条 spot-check：

```bash
python -c "from engine import NllbTranslator; t=NllbTranslator(); t.load(); print(t.translate('含新词的测试句', pro_mode=True).text)"
```

5. 验收通过后导出下一批：`--batch-index 2`

---

## 去重规则（脚本已做 + 合并时注意）

| 层级 | 规则 |
|------|------|
| 导出时 | 同一英文小写只出现一次；已存在于 sqlite + overrides 的跳过 |
| AI 填表时 | en 不要重复；短语优先于单词 |
| 写入 overrides 时 | 若 en 已存在，用新行覆盖旧译法（build 时 overrides 优先级最高） |

---

## 主表 vs 完整库（为什么主表看起来「少」）

```
user_overrides.csv     ← 你只管这一小份（人工精品）
        +
UCS 三源 CSV         ← 自动合并（MyKeyWord 374 + UCS 972 类 + 同义词…）
        ↓
audio_glossary.sqlite  ← 程序实际查的（~10900 条）
```

主表不是完整词典，而是 **「最终裁决层」**。
