# 翻译系统是怎么工作的（大白话版）

> 给非程序员看。讲清楚：现在翻译用到哪些文件、每个文件干嘛、一句中文是怎么一步步变成英文音效名的。

---

## 一、这套东西到底在做什么

把你打的**中文**（或中英混合）变成**BOOM 风格的英文音效名**，比如：

- `喷火器` → `Flamethrower`
- `奥古斯塔 A109 飞走 01` → `Augusta A109 Fly Away 01`
- `金属门重重关上` → `Metal Door Heavily Close`

还能做：英文→中文、中文整句→英文（写描述用）、英文长句→中文。全部**离线**，不联网。

---

## 二、翻译时用到哪些文件，各自干嘛（大白话）

把整个流程想象成一条流水线，每个文件是一个工位：

| 工位（文件） | 大白话作用 | 打个比方 |
|--------------|------------|----------|
| `translator/api.py` | **总入口**。你把中文交给它，它返回结果，并决定走哪条路（音效名 / 句子翻译） | 前台接待 |
| `translator/fxname_mode.py` | **核心流程指挥**。把下面几个工位按顺序串起来 | 流水线领班 |
| `translator/segment.py` + `data/jieba_userdict.txt` | **断词**。把「喷火器」当成一个词，而不是拆成「喷/火/器」 | 先把句子断成词 |
| `translator/overrides.py` + 三张表（见下） | **优先译法表**。常用词、易错词的「标准答案」 | 先查内部规定的标准译法 |
| `translator/cedict.py` + `glossary/sources/cc-cedict/cedict_ts.u8` | **通用字典兜底**。12 万词条的中英大字典 | 查《新华字典》 |
| `translator/nllb.py` + `engine.py` + `nllb_int8_model/` | **本地 AI 翻译机**。前面都查不到时才用 | 实在不会就问翻译机 |
| `translator/boom_snap.py` + `glossary/boom_style_index.sqlite` + `data/zh_en_alignment.csv` | **对照 BOOM 大词库改写**。把翻出来的词改成 BOOM 里真正常用的地道写法 | 对照行业惯例润色 |
| `glossary/fx_slots.py` | **排版**。首字母大写、按顺序拼成最终名字 | 最后排版定稿 |

### 「优先译法表」其实有三张（优先级从高到低）

| 表 | 谁来维护 | 作用 |
|----|----------|------|
| `translator/data/fx_overrides.csv` | **人手工维护（很小）** | 自动表偶尔选错时，手动纠正一行 |
| `fxengine/data/canonical_tokens.csv` | 受治理的主表（**只读，不改**） | 项目正式认证过的译法 |
| `translator/data/fx_overrides_auto.csv` | **机器自动生成（4.8 万条）** | 按 BOOM 词频自动挑最合适的译法，不用人堆 |

### 句子翻译（英→中 / 中→英整句）另外用到

| 文件 | 作用 |
|------|------|
| `engine.py` + `nllb_int8_model/` | 本地 AI 翻整句 |
| `glossary/audio_glossary.sqlite` | 翻译时保护专业术语不被翻错 |

### 背后的「参考资料库」（不直接翻译，但是质量来源）

| 文件 | 作用 |
|------|------|
| `glossary/boom_style_index.sqlite` | **BOOM 全库词频**（8 万条音效名、36 万短语）。判断一个词「在 BOOM 里常不常见」 |
| `docs/训练数据/`（本地，不进仓库） | BOOM 原始 Excel（115 个文件、约 10 万行），上面那个词频库就是从这里建的 |
| `docs/boom_mining/..._candidates.csv` | BOOM 中英对照挖出来的「中文词→英文词」证据，用来防止改错意思 |

## 三、一句中文是怎么一步步变成英文名的（核心逻辑）

以输入 `金属门重重关上` 为例：

1. **断词**：切成 `金属门` / `重重` / `关上`（虚词如「的、了、在」会被直接扔掉）。
2. **逐词翻译**，每个词按这个顺序找答案：
   - 先查**优先译法表**（手工 → 受治理表 → 自动表）；
   - 查不到再查**通用字典**（CC-CEDICT）；
   - 还查不到，才用**本地 AI 翻译机**（NLLB）。
   - 于是：`金属门→Metal Door`、`重重→heavily`、`关上→close`。
3. **对照 BOOM 大词库改写**（这是最关键的「翻得像」环节）：
   - 这个词在 BOOM 里**很常见** → 保留；
   - **不常见但有更地道的写法** → 换成 BOOM 的写法（例：`flame thrower`→`Flamethrower`）；
   - **不常见又没有靠谱替换** → 保留原样，不硬改。
4. **排版**：首字母大写、按输入顺序拼接。
5. **输出**：`Metal Door Heavily Close`。

一句话总结这套逻辑：

> **字典/AI 负责「翻得出来」，BOOM 大词库负责「翻得像」，自动表负责「不用人天天调」。**

## 四、想改翻译结果时，改哪里（不用懂代码）

- 某个词翻错了 → 在 `translator/data/fx_overrides.csv` 加一行：`中文,英文,,备注`，保存即可（它优先级最高）。
- 想看某句为什么翻成这样 → 运行 `python tools/quick_fxname_smoke.py "你的中文"`，它会把每个词的译法和决策一条条打出来。
- 想给整个系统打分 → 运行 `python tools/eval_translator.py`，会和 BOOM 真实英文对照算分。

更细的操作手册见 [TRANSLATOR_METHODOLOGY.md](TRANSLATOR_METHODOLOGY.md)；技术架构见 [TRANSLATOR_ARCHITECTURE.md](TRANSLATOR_ARCHITECTURE.md)。
